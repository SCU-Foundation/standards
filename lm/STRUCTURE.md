## Structural Rules

- all admin files MUST exist at the root of the unit folder  
- all normative content MUST be in a single canonical file named:  
  LM-v1.0.md  
- HTML editions MUST be placed in /html/  
- supporting assets MUST be placed in /assets/  
- no additional folders may be added without governance approval  
- all references MUST use repo‑scoped paths  
- all files MUST follow canonical formatting and naming conventions
